## Project Workflow
All changes (features, updates, bug fixes) follow this lifecycle:
1  **before**: read and clear current-state.md
2. **Planning**: Task is defined, scoped, and added to the `current-state.md` tracking list.
3. **Development**: Code is written, implemented, and local verification is performed.
4. **Testing**: Comprehensive functional, integration, and regression tests are executed.
5. **Deployment**: Changes are verified in the target environment and finalized.

Progress is tracked by moving tasks through these stages within this document.
